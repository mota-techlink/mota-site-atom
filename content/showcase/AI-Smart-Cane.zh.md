### 挑战：为老年用户构建安全、连接与陪伴的桥梁

对于行动不便的老年人而言，拐杖已是不可或缺的辅助工具。但除了物理支撑，这个群体还面临三重复合难题，传统辅助设备对此束手无策。

*   **紧急定位：** 当老人摔倒或在陌生环境中迷路时，家属和护理人员缺乏可靠的方式快速找到他们。
*   **数字鸿沟：** 许多老年用户的手部灵活性、视力或数字素养不足以操作智能手机。语音是最自然的交互方式，但需要一个随时在手且始终在线的设备。
*   **认知参与与情感陪伴：** 孤独感与认知退化是老年人面临的严重健康风险。被动的辅助设备无法提供任何对话陪伴或认知刺激。

挑战在于：如何将上述所有能力嵌入用户本便持握的设备——拐杖——中，同时不给终端用户增加任何技术复杂度。

### 解决方案：灵机 AI Agent 平台上拐杖


#### 硬件架构

设备将定制化硬件与 MOTA 云边软件层深度结合：

| 组件 | 功能 |
|---|---|
| **语音芯片** | 设备端音频采集与播放硬件 |
| **利尔达 4G 模块** | 始终在线的蜂窝网络连接，支持云端推理 |
| **北斗定位芯片** | 实时卫星 GPS 位置追踪 |
| **AI 按键** | 一键按下即说的交互方式——无需屏幕，无需 APP |

#### 交互流程：按键说话

整套用户体验被简化为单一物理手势，针对老年人的运动能力进行了专项优化：

1. **长按** AI 按键 → 麦克风激活，用户自然说话。
2. **松开**按键 → 语音输入通过 4G 网络流式传输至云端大模型。
3. 大模型处理查询并将合成语音响应流回设备。
4. 用户通过内置扬声器听到回复。

**打断支持：** 若大模型正在播报回复时用户再次长按按键，播放立即静音，系统随即准备接收新的语音查询。这彻底消除了令人沮丧的"无法跳过"体验。

#### 大模型接入：中移动大模型

MOTA 的 Agent 平台新增了**中移动大模型**连接器，企业客户可在 Agent 基础配置中选择其作为底层推理模型，从而实现：

*   **自然对话**，超越简单关键词识别——模型能理解老年用户常用的省略式、口语化表达。
*   **垂类知识能力**，通过 Agent 配置接入特定领域知识，如健康提醒、用药时间或本地天气。
*   **会话内多轮记忆**，支持自然流畅的连续对话。

#### Agent API 接口层

MOTA 为南京立奇硬件平台设计了两个清晰的集成接口：

*   **ASR（语音识别）：** 接收麦克风采集的原始音频流，返回文本供大模型推理。
*   **TTS（语音播报）：** 接收大模型的文本响应，将合成语音流与播放指令一同返回至拐杖扬声器。

简洁的 API 边界使硬件团队与 AI 平台团队可以独立演进，降低集成复杂度，加速未来迭代。

### 方案价值

通过将语音 AI、卫星定位与大模型 Agent 嵌入一款日常使用的医疗辅助设备，该方案在三个维度上实现了可量化的价值：

*   **安全保障：** 北斗 GPS 支持家属或护理人员实时共享位置，实现紧急情况的快速响应。
*   **无障碍使用：** 仅需按下一个按键——无需智能手机，无需 APP，零学习成本。
*   **情感陪伴：** 自然语言大模型对话为独居或低交互环境中的老年用户提供认知刺激与情感连接。
