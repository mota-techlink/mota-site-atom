## 概要

生成 AI の時代において、企業は自社固有のドメイン知識を理解し、データプライバシーを確保し、低レイテンシで動作する<Gradient from="red" to="orange">インテリジェントアシスタント</Gradient>を必要としています。標準的な「パッケージ型」ソリューションは、高額な API コスト、遅い応答時間、データのハルシネーション問題に直面することが多くあります。

**MOTA TECHLINK** は、**検索拡張生成（RAG）**を活用してプライベートな企業データに基づく正確でコンテキストを認識した回答を提供する、フルスタックのエッジネイティブ AI ソリューションを設計しました。従来の重量級サーバーを回避し、Cloudflare のグローバルエッジネットワークを活用することで、比類のないパフォーマンスとコスト効率を実現しています。

## 課題

静的サイトアーキテクチャ（Hugo）上にプロダクションレディな AI チャットボットを構築するにあたり、いくつかのユニークなエンジニアリング課題に直面しました：

1.  **エッジランタイム互換性：** Auth.js などの標準的な Node.js 認証ライブラリは、V8 エッジランタイムでは利用できない API に依存しています。
2.  **状態管理：** サーバーレスかつステートレスな環境で、競合状態を発生させることなく OAuth 2.0 フローと会話履歴を処理する必要があります。
3.  **リアルタイム体験：** 非同期のデータベース書き込みによる履歴監査を同時に実行しながら、スムーズなタイプライター風のストリーミングレスポンス（SSE）を実現する必要があります。
4.  **コストと精度：** 高品質な LLM 推論の要件と厳格な運用コスト管理のバランスを取る必要があります。

## ソリューション：エッジネイティブアーキテクチャ

> 単純な API 呼び出しを超え、Cloudflare のモダンな技術スタックを使用して堅牢な**サーバーレス AI ゲートウェイ**を構築しました。

![](/images/showcase/chatbot/edge-solution.webp)

### システムアーキテクチャ

* **フロントエンド：** Hugo（静的）+ ネイティブ JS（SSE パーサーおよび Markdown レンダラー）。
* **エッジ API ゲートウェイ：** Cloudflare Pages Functions（V8 ランタイム）。
* **認証：** **Workers KV** で保護されたカスタムネイティブ OAuth 2.0 フロー（Google）。
* **ナレッジベース（RAG）：** **Cloudflare Vectorize**（ベクトルデータベース）+ **Workers AI**（埋め込みモデル）。
* **推論エンジン：** Workers AI（エッジ GPU）上で稼働する **Llama 3**。
* **永続化：** **Cloudflare D1**（サーバーレス SQL）によるユーザープロファイルとチャット履歴の管理。

### コア機能

#### 1. 検索拡張生成（RAG）
汎用チャットボットとは異なり、当システムは回答前に MOTA TECHLINK の独自ドキュメントを「読み込み」ます。**AutoRAG** ワークフローを活用して関連コンテキストの埋め込み、インデックス作成、検索を行い、回答の正確性とハルシネーションの排除を保証します。

#### 2. ネイティブ OAuth 2.0 実装
エッジランタイムの制約を克服するため、軽量なネイティブ OAuth 認証フローを設計しました。
* **セキュリティ：** **Workers KV** を使用した状態検証により CSRF 攻撃を防止。
* **セッション管理：** セキュアな HTTP-only JWT セッションを **D1** に保存し、エッジで 10ms 未満のレイテンシでユーザー認証を検証。

#### 3. リアルタイムストリーミングと履歴管理
カスタムの**サーバー送信イベント（SSE）**プロトコルを実装しました。
* **ノンブロッキング I/O：** システムは AI レスポンスをバイト単位でユーザーにストリーミングし、即座のフィードバックを実現。
* **非同期ログ：** `Response.tee()` と `context.waitUntil` を活用し、会話履歴は D1 データベースに非同期で永続化され、ユーザー体験にレイテンシを追加しません。

## 技術的ハイライト

### エッジ互換性の課題を解決
標準的な認証ライブラリはエッジ環境では動作しません。OAuth コールバックロジックを書き直し、標準の `fetch` API を使用してトークン交換と OpenID Connect ユーザー情報の取得を手動で処理することで、Cloudflare V8 ランタイムとの 100% の互換性を確保しました。

### コストとパフォーマンスの最適化
「知識注入」方式（プロンプトにすべてのドキュメントを送信）から RAG アーキテクチャに切り替えることで、**入力トークン消費量を 90% 削減**しました。さらに、Cloudflare Workers AI を活用してエッジ GPU 上で推論を実行することにより、高価な常時稼働 GPU サーバーの必要性を排除しました。

## サポートを超えて：生涯学習資産
![](/images/showcase/chatbot/lifegrowth.webp)

この技術の真の価値は、**24 時間 365 日の多言語カスタマーサポート**の提供をはるかに超えています。その核心的な強みは**進化する能力**にあります：

* **継続的改善サイクル：** すべてのインタラクションが D1 データベースに安全に記録されます。
* **能動的学習：** 管理者は会話ログを確認し、AI の回答品質を評価し、ナレッジベースのギャップを特定できます。
* **生涯成長：** 修正された情報や新しいビジネスケースを RAG システムに継続的にフィードバックすることで、AI のナレッジベースは有機的に成長します。単に質問に答えるだけでなく、よりスマートに、より正確に、そしてビジネス目標により整合するようになります。
