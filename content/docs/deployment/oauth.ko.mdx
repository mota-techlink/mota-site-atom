## Google 측
1. Google Cloud 프로젝트 생성: [Google Cloud Console](https://console.cloud.google.com/)을 방문하여 새 프로젝트를 생성합니다.

2. OAuth 동의 화면 설정: "APIs & Services" -> "OAuth consent screen"에서 "External"을 선택하고 필요한 정보를 입력합니다.

3. [자격 증명](https://console.cloud.google.com/apis/credentials) 생성: "Credentials" 페이지에서 "Create credentials" -> "OAuth client ID"를 클릭합니다.
![Create OAuth Client ID](/images/docs/CreateOAuthClientID.webp)

- 애플리케이션 유형으로 "Web application"을 선택합니다.

- Authorized JavaScript origins: `http://localhost:3000`을 추가합니다.

- Authorized redirect URIs: 로컬 Supabase 콜백 URL을 추가합니다. 형식은 `http://127.0.0.1:54321/auth/v1/callback`입니다.

4. Client ID와 Secret 받기: 성공적으로 생성되면 Client ID와 Client Secret을 받게 됩니다.

5. 로컬 Supabase 설정: 로컬 Supabase Studio (`http://127.0.0.1:54323`)를 열고, "Authentication" -> "Providers"에서 "Google"을 활성화하고 방금 받은 Client ID와 Client Secret을 입력합니다.


## 로컬 Supabase 설정

Supabase를 로컬에서 사용할 때, Authentication의 타사 로그인 프로바이더(Google, GitHub 등)는 로컬 Studio UI에서 직접 활성화할 수 없습니다.

프로젝트의 Supabase 설정 파일을 편집하여 활성화해야 합니다.

로컬 Supabase Auth Providers를 활성화하는 자세한 단계는 다음과 같습니다:

**핵심 단계: config.toml 파일 수정**
로컬 Supabase의 모든 설정은 프로젝트 루트 디렉토리의 `supabase/config.toml` 파일에 있습니다. 이 파일에서 해당 OAuth 프로바이더 설정 블록을 찾거나 추가하고 Client ID와 Secret을 입력해야 합니다.

1. 설정 파일 찾기
프로젝트를 열고 다음 파일을 찾습니다: `프로젝트 루트 디렉토리/supabase/config.toml`

2. 타사 프로바이더에서 자격 증명 받기
설정하기 전에 먼저 활성화하려는 타사 서비스 프로바이더(Google Cloud Console, GitHub Developer Settings 등)에서 애플리케이션을 등록하고 다음 두 가지 핵심 정보를 받아야 합니다:

- Client ID

- Client Secret

로컬 개발 시 중요 참고: 타사 애플리케이션의 "Authorized redirect URIs"를 설정할 때, 로컬 Supabase의 기본 콜백 주소는 보통 다음과 같습니다: `http://127.0.0.1:54321/auth/v1/callback` (supabase를 시작할 때 표시되는 API URL 포트와 포트 번호가 일치하는지 확인하세요)

3. config.toml 편집
`config.toml` 파일에서 아래로 스크롤하여 `[auth.external]` 관련 영역을 찾습니다. 없으면 수동으로 추가해야 합니다.

예시: Google 로그인 활성화

파일에 다음 내용을 추가하거나 수정하고, `your_client_id`와 `your_client_secret`을 실제로 받은 값으로 교체합니다:

```Ini, TOML
# supabase/config.toml

# ... 기타 설정 ...

[auth.external.google]
enabled = true
client_id = "Your Google Client ID"
secret = "Your Google Client Secret"
# redirect_uri는 로컬에서 명시적으로 설정할 필요가 없으며, Supabase가 자동으로 처리합니다.
# 단, Google 콘솔에서 올바른 localhost 콜백 주소를 설정했는지 확인하세요.

# 예시: GitHub 로그인 활성화
# [auth.external.github]
# enabled = true
# client_id = "Your GitHub Client ID"
# secret = "Your GitHub Client Secret"

# ... 기타 설정 ...
```
`enabled = true`인지 확인하세요.

4. 로컬 Supabase 재시작
설정 파일을 수정한 후, 변경 사항이 적용되려면 로컬 Supabase 서비스를 반드시 재시작해야 합니다.

터미널에서 실행:

```Bash
npx supabase stop
npx supabase start
```
5. 확인
재시작 후, 로컬 Supabase Studio 패널(보통 `http://127.0.0.1:54323`)을 다시 방문합니다.

Authentication -> Providers 페이지로 이동합니다. 설정 파일에서 추가한 프로바이더(예: Google)가 이미 "Enabled"로 표시되어 있어야 하며, 확장하면 설정한 Client ID를 확인할 수 있습니다.
