## 개요

검색 엔진 최적화(SEO)는 검색 엔진에서 웹사이트의 가시성을 높이는 데 매우 중요합니다. 이 가이드에서는 Next.js 애플리케이션에 Google Analytics와 Google Search Console을 설정하는 방법을 설명합니다.

## 환경 변수

SEO 기능을 활성화하려면 `.env` 또는 `.env.local` 파일에 다음 환경 변수를 설정해야 합니다:

```bash
NEXT_PUBLIC_GA_MEASUREMENT_ID=G-XXXXXXXXXX
NEXT_PUBLIC_GOOGLE_SITE_VERIFICATION=your-verification-code
```
## Google Analytics (NEXT_PUBLIC_GA_MEASUREMENT_ID)
Google Analytics 4 (GA4)는 방문자가 웹사이트와 어떻게 상호작용하는지 이해하는 데 도움을 줍니다.

### 측정 ID 받는 방법:
1. [Google Analytics](https://analytics.google.com/)에 접속하여 Google 계정으로 로그인합니다.
2. 왼쪽 하단의 관리(톱니바퀴 아이콘)를 클릭합니다.
3. 속성 열에서 데이터 스트림을 클릭합니다.
4. 웹 데이터 스트림을 선택합니다. 없는 경우 스트림 추가 -> 웹을 클릭하여 생성합니다.
5. 웹 스트림 세부정보 창에서 측정 ID를 복사합니다 (보통 G-로 시작합니다).
6. 이 ID를 .env 파일에 NEXT_PUBLIC_GA_MEASUREMENT_ID로 붙여넣습니다.

### Google Search Console (NEXT_PUBLIC_GOOGLE_SITE_VERIFICATION)
Google Search Console 도구는 사이트의 검색 트래픽 및 성능을 측정하고, 문제를 수정하며, Google 검색 결과에서 사이트를 돋보이게 하는 데 도움을 줍니다.

### 인증 코드 받는 방법:
1. [Google Search Console](https://search.google.com/search-console)에 접속하여 로그인합니다.
2. 속성 추가를 클릭합니다 (이미 추가한 경우 드롭다운에서 속성을 선택합니다).
3. URL 접두사 방법을 선택하고 웹사이트 URL을 입력합니다 (예: https://your-domain.com). 계속을 클릭합니다.
4. 인증 방법 옵션에서 아래로 스크롤하여 HTML 태그를 찾습니다.
5. 제공된 메타 태그를 복사합니다. 다음과 같은 형태입니다:
`<meta name="google-site-verification" content="your-verification-code" />`
6. content 값(your-verification-code)만 추출합니다.
7. 이 코드를 .env 파일에 NEXT_PUBLIC_GOOGLE_SITE_VERIFICATION으로 붙여넣습니다.

### 설정 확인
이 변수를 추가하고 사이트를 배포한 후 (또는 로컬 서버를 재시작한 후):

- Google Analytics: 사이트를 방문하고 Google Analytics의 실시간 보고서에서 방문이 추적되고 있는지 확인합니다.
- Google Search Console: Google Search Console의 인증 페이지로 돌아가서 확인을 클릭합니다. 소유권이 확인되었다는 메시지가 표시됩니다.
