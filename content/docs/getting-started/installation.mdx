🛠️ Installation & Setup Guide
This guide will help you set up the Mota Atom development environment from scratch. The project is built on Next.js, Supabase, and Cloudflare Pages.

📋 Table of Contents
1. Environment
2. Project Initialization
3. Supabase Integration
4. Core Features
5. Deployment

## 1. Environment
It is recommended to use Conda to manage the project environment and isolate the Node.js version.

```bash
# Create an independent environment (specify Node.js 20)
conda create -n mota-atom nodejs=20 -y

# Activate environment
conda activate mota-atom
```

## 2. Project Initialization
Initialize the Next.js framework and the Shadcn UI component library.

```bash
# 1. Initialize Next.js project (TypeScript, Tailwind, ESLint)
npx create-next-app@latest mota-atom --typescript --tailwind --eslint

# Enter project directory
cd mota-atom

# 2. Initialize Shadcn UI
npx shadcn@latest init
```
💡 Tip: When configuring Shadcn UI, it is recommended to choose Slate or Gray as the color theme for the best visual effect.

## 3. Supabase Integration
This project uses Supabase as the backend service (database, authentication).

### 3.1 CLI Installation & Initialization
```bash
# Install Supabase CLI (dev dependency only)
npm install supabase --save-dev

# Verify installation (should show version number, e.g., 2.72.9)
npx supabase --version

# Initialize project (generates supabase folder)
npx supabase init
```

### 3.2 Start Local Environment (Local Docker)
Ensure your Docker is running, then start the local Supabase instance:

```bash
npx supabase start
```
After starting successfully, you will get the following local service addresses:

- Console (Studio): http://127.0.0.1:54323
- Mailpit (Email testing): http://127.0.0.1:54323/project/default

### 3.3 Install Dependencies
```bash
# @supabase/ssr is used for cookie handling in server components and middleware
npm install @supabase/supabase-js @supabase/ssr
```

### 3.4 Environment Variable Configuration
Create a `.env.local` file in the project root:

```bash
# Get from npx supabase status
NEXT_PUBLIC_SUPABASE_URL=http://127.0.0.1:54321
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-local-anon-key
```

### 3.5 Database Workflow
**Development**: When you modify the local database structure, use the following command to generate SQL migration files:

```bash
# Extract local changes to supabase/migrations directory
npx supabase db diff -f init_schema
npx supabase db diff -f add_user_table
```

**Production Sync**: Push local schema changes to the remote production database:

```bash
# Must run npx supabase login first
npx supabase db push
```

## 4. Core Features
### 🌓 Theme Support (Light/Dark Mode)
Integrate next-themes to support light/dark mode switching.

```bash
npm install next-themes
```

**Configuration**: Wrap `ThemeProvider` in `layout.tsx`. Shadcn components will adapt automatically.

### 🌐 Multi-language Support (i18n)
Use next-intl to implement internationalization.

```bash
npm install next-intl
```
- **Configuration Files**: `src/i18n.ts`, `src/middleware.ts`
- **Translation Files**: `src/messages/zh.json`, `src/messages/en.json`
- **Routing Structure**: Use `src/app/[locale]/` dynamic routing

### 📝 Blog System (Taxonomy Template)
Build a content engine based on MDX and Tailwind Typography.

```bash
# Install MDX parsing and metadata processing tools
npm install gray-matter next-mdx-remote
npm install -D @tailwindcss/typography
```
- **Configure Plugin**: Update `tailwind.config.ts` to include the typography plugin.
- **Read Engine**: `src/lib/mdx.ts`
- **Page Implementation**: `src/app/[locale]/(portal)/blog/page.tsx` (Resend style list)

## 5. Deployment
This project is optimized for Cloudflare Pages.

### 5.1 Adapter Installation
Install the Cloudflare adapter and lock the React version (to ensure compatibility):

```bash
npm install next@15.5.2 react@19 react-dom@19 --save-exact
npm install -D @cloudflare/next-on-pages
```

### 5.2 Build Project
Use the dedicated Cloudflare command to build:

```bash
npx @cloudflare/next-on-pages
```

### 5.3 Local Preview
Before pushing to GitHub, simulate the Cloudflare Edge environment locally for testing:

```bash
npx wrangler pages dev .vercel/output/static
```

### Appendix: Common Commands
- **Check Supabase Status**: `npx supabase status`
- **Reset Local Database**: `npx supabase db reset`
