# 安装与配置指南
本指南将帮助你从零开始搭建 Mota Atom 开发环境。项目基于 Next.js、Supabase 和 Cloudflare Pages 构建。

**目录**
- 环境准备
- 项目初始化
- 后端集成
- 核心功能
- 部署构建

## 1. 环境准备
推荐使用 Conda 管理项目环境，以隔离 Node.js 版本。

```bash
# 创建独立环境（指定 Node.js 20）
conda create -n mota-atom nodejs=20 -y
```

# 激活环境
conda activate mota-atom

## 2. 项目初始化
初始化 Next.js 框架及 Shadcn UI 组件库。

```bash
# 1. 初始化 Next.js 项目（TypeScript, Tailwind, ESLint）
npx create-next-app@latest mota-atom --typescript --tailwind --eslint

# 进入项目目录
cd mota-atom

# 2. 初始化 Shadcn UI
npx shadcn@latest init
💡 提示：在配置 Shadcn UI 时，建议颜色主题选择 Slate 或 Gray 以获得最佳视觉效果。
```

## 3. 后端集成（Supabase）
本项目使用 Supabase 作为后端服务（数据库、认证）。

### 3.1 CLI 安装与初始化
```bash
# 安装 Supabase CLI（仅开发依赖）
npm install supabase --save-dev

# 验证安装（应显示版本号，如 2.72.9）
npx supabase --version

# 初始化项目（生成 supabase 文件夹）
npx supabase init
```

### 3.2 启动本地环境（Local Docker）
确保你的 Docker 已经在运行，然后启动本地 Supabase 实例：

```bash
npx supabase start
启动成功后，你将获得以下本地服务地址：
```

Console（Studio）：http://127.0.0.1:54323

Mailpit（邮件测试）：http://127.0.0.1:54323/project/default

### 3.3 安装依赖
```bash
# @supabase/ssr 用于服务端组件和中间件中的 Cookie 处理
npm install @supabase/supabase-js @supabase/ssr
```

### 3.4 环境变量配置
在项目根目录创建 .env.local 文件：

```Code snippet
# 获取自 npx supabase status
NEXT_PUBLIC_SUPABASE_URL=http://127.0.0.1:54321
NEXT_PUBLIC_SUPABASE_ANON_KEY=你的本地AnonKey
```

### 3.5 数据库工作流
开发阶段：当你修改了本地数据库结构后，使用以下命令生成 SQL 迁移文件：

```bash
# 将本地变更提取到 supabase/migrations 目录
npx supabase db diff -f init_schema
npx supabase db diff -f add_user_table
```
生产同步：将本地的表结构变更推送到远程生产数据库：

```bash
# 需先运行 npx supabase login
npx supabase db push
```

## 4. 核心功能

### 🌓 主题支持（明暗模式）
集成 next-themes 以支持黑白模式切换。

```bash
npm install next-themes
```

配置：请在 layout.tsx 中包裹 ThemeProvider。Shadcn 组件会自动适配。

### 🌐 多语言支持（i18n）
使用 next-intl 实现国际化。

```bash
npm install next-intl
```
配置文件：src/i18n.ts、src/middleware.ts

翻译文件：src/messages/zh.json、src/messages/en.json

路由结构：使用 src/app/[locale]/ 动态路由

### 📝 博客系统（Taxonomy 模板）
基于 MDX 和 Tailwind Typography 构建内容引擎。

```bash
# 安装 MDX 解析与元数据处理工具
npm install gray-matter next-mdx-remote
npm install -D @tailwindcss/typography
```
配置插件：更新 tailwind.config.ts 引入 typography 插件。

读取引擎：src/lib/mdx.ts

页面实现：src/app/[locale]/(portal)/blog/page.tsx（Resend 风格列表）

## 5. 部署构建
本项目针对 Cloudflare Pages 进行了优化。

### 5.1 适配器安装
安装 Cloudflare 适配器并锁定 React 版本（确保兼容性）：

```bash
npm install next@15.5.2 react@19 react-dom@19 --save-exact
npm install -D @cloudflare/next-on-pages
```

### 5.2 构建项目
使用 Cloudflare 专用命令进行构建：

```bash
npx @cloudflare/next-on-pages
```

### 5.3 本地预览
在推送到 GitHub 之前，在本地模拟 Cloudflare Edge 环境进行测试：

```bash
npx wrangler pages dev .vercel/output/static
```

- 重置本地数据库：`npx supabase db reset`
